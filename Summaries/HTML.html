1. HTML tags

  • Объявление <!DOCTYPE html> всегда должно быть первой строкой кода в ваших HTML-файлах. Это позволяет браузеру узнать, какую версию HTML ожидать.
  • Элемент <html> будет содержать весь ваш HTML-код.
  • Информация о веб-странице, например заголовок, находится внутри <head> страницы.
  • Вы можете добавить заголовок на свою веб-страницу, используя элемент <title> внутри заголовка. Название веб-страницы отображается на вкладке браузера.
  • Теги привязки <a> используются для ссылки на внутренние страницы, внешние страницы или контент на той же странице. Вы можете создавать разделы на веб-странице и переходить к ним, используя теги <a> и добавляя идентификаторы к элементам, к которым вы хотите перейти.
  • Пробелы между элементами HTML помогают упростить чтение кода, не изменяя при этом внешний вид элементов в браузере. Отступы также помогают облегчить чтение кода. Это делает отношения между родителями и детьми видимыми.
  • Комментарии пишутся в HTML с использованием следующего синтаксиса: <!-- комментарий -->.

2. HTML tables

  • Элемент <table> создает таблицу. Элемент <li> добавляет строки в таблицу.
  • Чтобы добавить данные в строку, вы можете использовать элемент <td>.
  • Заголовки таблиц поясняют значение данных. Заголовки добавляются с помощью элемента <th>.
  • Табличные данные могут охватывать столбцы с помощью атрибута colspan.
  • Табличные данные могут охватывать строки с помощью атрибута rowspan.
  • Таблицы можно разделить на три основные части: заголовок, тело и нижний колонтитул.
  • Заголовок таблицы создается с помощью элемента <thead>. Тело таблицы создается с помощью элемента <tbody>.
  • Нижний колонтитул таблицы создается с помощью элемента <tfoot>.
  • Все свойства CSS, о которых вы узнали в этом курсе, можно применять к таблицам и их данным.

3. HTML forms

  • Назначение <form> — позволить пользователям вводить информацию и отправлять ее.
  • Атрибут действия <form>strong> определяет, куда направляется информация формы.
  • Атрибут метода <form> определяет, как информация отправляется и обрабатывается.
  • Чтобы добавить поля для ввода информации пользователями, мы используем элемент <input> и устанавливаем атрибут type в поле по нашему выбору:
    • Установка типа на text создает поле с одной строкой для ввода текста.
    • Установка типа password создает поле с одной строкой, которое подвергает цензуре ввод текста.
    • Установка типа на number создает поле с одной строкой для ввода числа.
    • Установка типа range создает ползунок для выбора из диапазона чисел.
    • Установка типа checkbox создает один флажок, который можно связать с другими флажками.
    • Установка типа radio создает радиокнопку, которую можно связать с другими радиокнопками.
    • Установка типа на datalist соединит <input> с элементом <datalist>, если идентификаторы обоих совпадают.
    • Установка типа «отправить» создает кнопку отправки.
  • Элемент <select> заполняется элементами <option> и отображает выбор из раскрывающегося списка.
  • Элемент <datalist> заполняется элементами <option> и работает с <input> для поиска вариантов.
  • Элемент <textarea> — это поле ввода текста с настраиваемой областью.
  • При отправке <form> имя полей, которые принимают ввод, и значение этих полей отправляются в виде пар имя=значение.

4. Form validation

  • Проверка на стороне клиента происходит в браузере перед отправкой информации на сервер.
  • Добавление требуемого атрибута к элементу, связанному с вводом, проверяет, что поле ввода содержит информацию.
  • Присвоение значения атрибуту "min" элемента числового ввода проверяет приемлемое минимальное значение.
  • Присвоение значения атрибуту "max" элемента числового ввода проверяет приемлемое максимальное значение.
  • Присвоение значения атрибуту "minlength" элемента текстового ввода проверяет приемлемое минимальное количество символов.
  • Присвоение значения атрибуту "maxlength" элемента текстового ввода проверяет допустимое максимальное количество символов.
  • Присвоение "regex" регулярного выражения "pattern" сопоставляет ввод с предоставленным регулярным выражением.
  • Если проверка form не проходит, пользователь получает сообщение с объяснением причины, и form не может быть отправлена.

5. Semantic HTML

  • Семантический HTML придает значение странице с помощью определенных элементов, которые обеспечивают контекст относительно того, что находится между тегами.
  • Семантический HTML — это современный стандарт, который делает веб-сайт доступным для людей, которые используют программы чтения с экрана для перевода веб-страницы и улучшения SEO вашего веб-сайта.
  • <header>, <nav>, <main> и <footer> создают базовую структуру веб-страницы.
  • <section> определяет элементы в документе, такие как главы, заголовки или любую другую область документа с той же темой.
  • <article> содержит контент, который имеет смысл сам по себе, например, статьи, блоги, комментарии и т. д.
  • <aside> содержит информацию, связанную с основным содержанием, но не требуемую для понимания доминирующей информации.
  • <figure> инкапсулирует все типы носителей.
  • <figcaption> используется для описания медиа в <figure>.
  • Элементы <video>, <embed> и <audio> используются для мультимедийных файлов.
