1. Set the viewport

    <!DOCTYPE html>

    <html lang="en">

    <head>

    ...

    <meta name="viewport" content="width=device-width, initial-scale=1">

    ...

    </head>

    ...

  • Использование мета-тега области просмотра width=device-width указывает странице соответствовать ширине экрана
  • Некоторые браузеры сохраняют постоянную ширину страницы при повороте в ландшафтный режим и масштабируют, а не перекомпоновывают, чтобы заполнить экран. Добавление значения initial-scale=1 указывает браузерам устанавливать соотношение 1:1 между пикселями CSS и независимыми от устройства пикселями независимо от ориентации устройства и позволяет странице использовать всю ширину альбомной ориентации.
  • В дополнение к настройке начального масштаба вы также можете установить следующие атрибуты в области просмотра:
    • minimum-scale
    • maximum-scale
    • user-scalable
  • Если они установлены, они могут отключить возможность пользователя масштабировать область просмотра, что может вызвать проблемы с доступностью. Поэтому мы не рекомендуем использовать эти атрибуты.

2. Размер контента в Viewport

  • Изображение имеет фиксированные размеры, и если оно больше, чем область просмотра, это вызовет полосу прокрутки.

    Обычный способ решить эту проблему — установить для всех изображений максимальную ширину 100%. Это приведет к тому, что изображение сожмется, чтобы соответствовать имеющемуся пространству, если размер области просмотра будет меньше, чем изображение. Однако, поскольку максимальная ширина, а не ширина равна 100%, изображение не будет растягиваться больше своего естественного размера.

    Как правило, безопасно добавить следующее в таблицу стилей, чтобы у вас никогда не было проблем с изображениями, вызывающими полосу прокрутки.


    • img {

    max-width: 100%;

    display: block;

    }

  • При использовании max-width: 100% вы переопределяете естественные размеры изображения, однако вы все равно должны использовать атрибуты width и height в теге <img>. Это связано с тем, что современные браузеры будут использовать эту информацию для резервирования места для изображения перед его загрузкой, что поможет избежать смещения макета при загрузке содержимого.

3. Макет

  • Поскольку размеры и ширина экрана в пикселях CSS сильно различаются между устройствами (например, между телефонами и планшетами и даже между разными телефонами), контент не должен зависеть от конкретной ширины области просмотра для хорошей визуализации.

    Современные методы компоновки CSS, такие как Flexbox, Grid Layout и Multicol, значительно упрощают создание этих гибких сеток.

4. Медиа-запросы

  • Медиа-запросы размера Viewport

    • Медиа-запросы позволяют нам создавать отзывчивый интерфейс, в котором определенные стили применяются к маленьким экранам, большим экранам и где-либо между ними. Поэтому функция, которую мы здесь обнаруживаем, — это размер экрана, и мы можем проверить следующие вещи.
    • width (min-width, max-width)
    • height (min-height, max-height)
    • orientation
    • aspect-ratio
    Все эти функции отлично поддерживаются браузерами. Для получения более подробной информации, включая информацию о поддержке браузеров, см. width, height, orientation и aspect-ratio на MDN.
  • Медиа-запросы в зависимости от возможностей устройства

    • Учитывая диапазон доступных устройств, мы не можем сделать предположение, что каждое большое устройство представляет собой обычный настольный или портативный компьютер или что люди используют только сенсорный экран на небольшом устройстве. С некоторыми новыми дополнениями к спецификации медиа-запросов мы можем проверить такие функции, как тип указателя, используемого для взаимодействия с устройством, и может ли пользователь наводить курсор на элементы.
    • hover
    • pointer
    • any-hover
    • any-pointer
    Эти новые функции хорошо поддерживаются во всех современных браузерах. Узнайте больше на страницах MDN для hover, any-hover, pointer и any-pointer

5. Выбор точек останова

  • Не определяйте точки останова на основе классов устройств. Определение точек останова на основе конкретных устройств, продуктов, торговых марок или операционных систем, которые используются сегодня, может привести к кошмару обслуживания. Вместо этого сам контент должен определять, как макет приспосабливается к своему контейнеру.
    • Выберите основные контрольные точки, начав с малых размеров, а затем увеличивая
      Сначала разработайте контент так, чтобы он помещался на маленьком экране, а затем расширяйте экран до тех пор, пока не понадобится точка останова. Это позволяет оптимизировать точки останова на основе контента и поддерживать минимально возможное количество точек останова.
    • Затем измените размер браузера, пока между элементами не останется слишком много пустого пространства, и прогноз просто не будет выглядеть так хорошо.
    • Чтобы вставить точку останова на 600px, создайте два медиа-запроса в конце вашего CSS для компонента, один для использования, когда браузер имеет размер 600px и меньше, а другой — когда он шире 600px.

      • @media (max-width: 600px) {

      }

      @media (min-width: 601px) {

      }

    • Внутри медиа-запроса для максимальной ширины 600 пикселей добавьте CSS, который предназначен только для маленьких экранов. Внутри медиазапроса для минимальной ширины 601px добавьте CSS для больших экранов.
  • При необходимости выбирайте второстепенные точки останова
    • В дополнение к выбору основных точек останова при значительном изменении макета также полезно выполнять корректировку незначительных изменений. Например, между основными контрольными точками может быть полезно настроить поля или отступы элемента или увеличить размер шрифта, чтобы он выглядел более естественно в макете.
  • Точно так же для больших экранов лучше ограничить максимальную ширину контента, чтобы он не занимал всю ширину экрана.

6. Оптимизация текста

  • Классическая теория удобочитаемости предполагает, что идеальный столбец должен содержать от 70 до 80 символов в строке (от 8 до 10 слов на английском языке). Таким образом, каждый раз, когда ширина текстового блока превышает 10 слов, рассмотрите возможность добавления точки останова.
